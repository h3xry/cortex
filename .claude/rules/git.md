# Git Rules

## Commit Format
```
type(scope): message
```

Types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`, `ci`

## Push Workflow
```bash
git add <specific-files>
git commit -m "type(scope): message"
git pull --rebase origin main
git push origin main
```

## Commit Identity (CRITICAL)

```
GIT_AUTHOR_NAME  = h3xry
GIT_AUTHOR_EMAIL = h3xry@users.noreply.github.com
GIT_COMMITTER_NAME  = h3xry
GIT_COMMITTER_EMAIL = h3xry@users.noreply.github.com
```
**EVERY git commit MUST set these environment variables** to ensure correct author/committer:
```bash
GIT_AUTHOR_NAME="h3xry" GIT_AUTHOR_EMAIL="h3xry@users.noreply.github.com" \
GIT_COMMITTER_NAME="h3xry" GIT_COMMITTER_EMAIL="h3xry@users.noreply.github.com" \
git commit -m "type(scope): message"
```
Never rely on git config alone - always pass env vars explicitly.

## Rules
- Never force push to main
- Never `git add -A` (use specific paths)
- Always pull --rebase before push
- No AI signatures in commits

## AI Signature Check (CRITICAL)

### Banned Patterns
```
🤖 Generated with [Claude Code]
Co-Authored-By: Claude
Co-Authored-By:.*anthropic
Co-Authored-By:.*openai
Generated by AI
Generated by Claude
Generated by GPT
\[AI\]
\[Claude\]
```

### Post-Commit Verification
After EVERY commit, run:
```bash
git log -1 --format="%B" | grep -iE "(claude|anthropic|openai|generated|co-authored.*ai|🤖)"
```

If match found → STOP and fix:
```bash
git commit --amend -m "type(scope): clean message"
```

### Why This Matters
- AI signatures expose workflow to external parties
- Some organizations have policies against AI-generated code
- Keeps commit history clean and professional

## Issue Title Format
- `plan:` - Planning documents
- `test:` - Test sub-issues
- `context:` - Session context (only 1 open)
- `bug:` / `feature:` - Standard issues
